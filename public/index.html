Performance-Monitor

The idea of this project born because I was courios about how to use JavaScript in multiples threads, it's mainly developed with Nodejs & Socket.io and with some React. I also used Redis to manage the uses of cores and be able to use javascript in every single thread.

How it works?
It gets information of the hardware performance (obtained with node methods) once per second, then comáre it with the previous information (if already exist), then result is emitted with Socket to be rendered with React.

Was a big challenge for me & it tooks me more than I expected, but I'm reallyy happy with what I learned working on it & with the result.

WIPIT-Board

I worked on the first MVP of WIPIT-Board with Diego Caplan as supervisor & product owner. Was my first time working in a team. We worked using the Agile methodology & we achieve an excellent product.
Wipit it's a backlog where developers can help each others. The backend was developed mainly with Nodejs & Express, we used PostgreSQL as db, also we developed a notification system & a chat in real time with Socket.io. In the front we used JavaScript & Chakra.ui for the styles. An authentication system was also developed but I didn't work on it.

I'm really glad to have been part of that team, it was an incredible experience.
